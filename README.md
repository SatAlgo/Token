# 🍵 Tea Shop Self-Service Token System

A self-service ordering + **digital token** system for a tea shop, built in
**Python (FastAPI)**. Customers order and pay on their phone; they get a token
with a **live animation** that can't be faked with a screenshot. Waiters verify
the token (visually or by scanning its signed QR) and mark it **served**, which
revokes it server-side so it can't be reused.

Solves two real problems: printer/power failures forcing fake-able paper tokens,
and cash handling at the counter.

---

## ✨ Features

### Customer (mobile)
- **Self-service menu + cart** with item photos, prices always re-checked server-side.
- **Table number** entered at checkout, shown on the token, waiter board & admin.
- **Online payment** — Razorpay (UPI/cards/wallets) with a free **demo mode** so
  it runs with zero setup and no real money.
- **Signed JWT tokens** — tampering with items/total breaks the signature.
- **Live anti-screenshot token** — a **blinking band that changes colour every
  second** (like the PMPML bus-ticket app) plus a live clock, so a static
  screenshot is obviously fake.
- **QR verification** — waiter scans the QR; server checks the signature *and*
  that the token hasn't already been served.
- **PWA + offline** — installable; a paid token stays visible even if wifi/power
  drops after issuance (service worker + localStorage cache).

### Waiter (mobile)
- **Own login account** (username + password) — created by the admin, separate
  from the admin login. Passwords are hashed (PBKDF2), never stored in plain text.
- **Realtime board** — paid orders pop in instantly over WebSockets, with table number.
- **QR / paste verification** and **one-tap "served"** that revokes the token
  (`jti` cleared) so it can't be reused.

### Admin (responsive — desktop & mobile)
- **Menu management** — add / edit / delete items, set prices, change category &
  emoji, **upload photos**, and **mark items unavailable** (sidebar on desktop,
  bottom-nav on mobile).
- **Complete order view** with **Pending / Completed** filters, showing what was
  ordered, the table, and whether it was delivered; mark delivered inline.
- **Analytics** — revenue today/week/all-time, avg order value, pending count,
  a 7-day revenue **bar chart** (Chart.js) and **top sellers**.
- **Image uploads via Cloudinary** (free tier), with a local-storage fallback.
- **Staff management** — create waiter accounts, set/reset their passwords,
  enable/disable, or delete them (Admin → Staff).

### Everywhere
- **Light / dark theme toggle** (🌙/☀️) on every screen, remembered across visits.

## 🧱 Tech stack (all free)

| Layer | Tech |
|---|---|
| Backend | FastAPI + Uvicorn |
| Database | SQLite via SQLModel |
| Realtime | FastAPI WebSockets |
| Token security | PyJWT (HS256) |
| Payments | Razorpay test mode / built-in demo mode |
| Frontend | Jinja2 + Tailwind (CDN) + vanilla JS + Chart.js |
| Images | Cloudinary (free tier) + local fallback |
| Offline | PWA service worker |

## 🚀 Run locally

```bash
cd Token
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env               # optional — defaults work as-is
uvicorn app.main:app --reload
```

Open <http://localhost:8000>.

| Screen | URL | Notes |
|---|---|---|
| Customer menu | `/` | Order & pay |
| Token | `/t/<order-id>` | Shown after payment |
| Admin | `/admin/login` | Owner login with `ADMIN_PASSWORD` (`tea123`) |
| Waiter board | `/login` | Waiter logs in with a username + password the admin created |
| API docs | `/docs` | Auto-generated by FastAPI |

**Logins are separate.** The **owner/admin** logs in at `/admin/login` with
`ADMIN_PASSWORD`. **Waiters** have their own accounts: the admin creates them in
**Admin → Staff** (sets each waiter's username & password), and waiters log in at
`/login`. Admins can do everything a waiter can; waiters can't reach admin pages.

**Try the full flow:** add items → **Pay** (demo mode confirms instantly) → you
land on the live token page. Open `/waiter` in another tab → the order appears →
**Mark served** → the customer token flips to **SERVED**.

## 💳 Switching on real payments (still free)

1. Make a free Razorpay account, grab **Test Mode** keys.
2. In `.env`: set `PAYMENT_MODE=razorpay`, `RAZORPAY_KEY_ID=...`,
   `RAZORPAY_KEY_SECRET=...`.
3. Restart. Checkout now opens real Razorpay (test cards/UPI, no real money).

## 🖼️ Item image uploads (Cloudinary, free)

Admin can upload a photo when adding/editing a menu item.

- **With Cloudinary:** set `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`,
  `CLOUDINARY_API_SECRET` in `.env` (free account at <https://cloudinary.com>).
  Images are hosted on Cloudinary's CDN.
- **Without it:** uploads are saved locally under `static/uploads/` and served by
  the app — zero setup, works offline. (On Render's ephemeral disk these reset on
  redeploy, so use Cloudinary in production.)

> **Note:** this version added columns to the database. If you ran an earlier
> version, delete `teashop.db` once so the new schema (item images, table number)
> is created.

## ☁️ Deploy free on Render

1. Push this folder to a GitHub repo.
2. Render → **New + → Blueprint** → select the repo (`render.yaml` is detected).
3. Set `ADMIN_PASSWORD` in the dashboard. Done.

> Render's free tier uses an ephemeral disk, so the SQLite file resets on
> redeploy. Fine for a pilot/demo; for persistence add a free Postgres instance
> and set `DATABASE_URL`.
